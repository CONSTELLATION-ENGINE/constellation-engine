CONSTELLATION

Constellation is an Autonomous Agent

Before the engine takes its first action, here is exactly what it will and will not do on your behalf. Acknowledge once to continue.
📁
Read & write inside its own folder
All databases, logs, models, and identity files live under <userData>/Constellation/engine/. The engine never reads or writes outside this folder.
🌐
Outbound HTTPS to your LLM provider(s)
Conversation turns and (optionally) model downloads talk to the providers you configured in setup. No telemetry. No analytics. No phone-home.
⚙️
Background processes
Spawns the engine plus an optional Mímir companion process. Both stop cleanly when you quit the launcher — no leftover daemons.
🔧
Execute tools the LLM authors during turns
File read / write / grep, web fetch, allowlisted shell commands. All restricted to the engine folder. No system-level access.
What this does NOT do
  • Read your home directory, Documents, or Desktop
  • Install or modify system services / firewall / startup entries
  • Send your data anywhere except your configured LLM provider(s)
  • Auto-update without asking you first
